# ОТВЕТЫ (краткие подсказки/решения)

1. `a=np.arange(1,11); a.shape -> (10,); a.dtype -> int64`
2. `X = np.array([2,4,6,8,10]).reshape(-1,1)`
3. `a * 3` — это векторизация, работает поэлементно.
4. `pd.DataFrame({"часы":[1,2,3,4], "оценка":[40,55,65,80]})`
5. `X = df[["часы"]]` — двойные скобки дают DataFrame (2D), одинарные — Series (1D).
6. `model.fit(X, df["оценка"]); model.predict([[2.5]])` → примерно 56–57.
7. w ≈ 13 — каждый час даёт ~13 баллов; b ≈ 27 — база без подготовки.
8. `plt.scatter(...)` + `plt.plot(X, model.predict(X))`.
9. Цена авто — regression (число). Клик — classification (да/нет).
10. MAE = (|10−12|+|20−18|+|30−33|)/3 = (2+2+3)/3 ≈ 2.33.
11. Должно совпасть с ручным расчётом.
12. (16,1), (4,1), (16,), (4,).
13. (3, 1) — 3 строки, 1 колонка.
14. `df["сдал"] = (df["оценка"] >= 60).astype(int)` → [0,0,1,1].
15. 2 часа → скорее всего 0; 5 часов → 1.
16. Два числа — вероятность класса 0 и класса 1, в сумме 1.0.
17. 4 совпадения из 5 → accuracy = 0.8.
18. `confusion_matrix(y_true, y_pred)` → [[2,0],[1,2]].
19. Например: цена квартиры, время доставки (regression); спам, отток клиента (classification).
20. Да, coef_ ≈ 3, intercept_ ≈ 7 (с небольшим отклонением из-за шума).
21. Используй mean_absolute_error, mean_squared_error (RMSE = sqrt(MSE)), r2_score.
22. mean ≈ 0, std ≈ 1 по каждой колонке.
23. Чтобы информация из test не "протекла" в обучение (data leakage); test должен быть полностью незнакомым.
24. depth=20: train R² ≈ 1.0, test заметно хуже → переобучение. depth=1 — недообучение.
25. 3 колонки: 1 (bias), x, x². PolynomialFeatures добавляет степени.
26. При alpha=100 coef_ заметно меньше — штраф прижимает веса к нулю.
27. accuracy = 0.95 (выглядит отлично), recall = 0 (модель не нашла НИ ОДНОЙ единицы). Recall честнее.
28. precision(1)=1.0, recall(1)=0.5, F1(1)≈0.67.
29. `def sigmoid(z): return 1/(1+np.exp(-z))` — S-образная кривая от 0 до 1.
30. w и b должны почти совпасть со sklearn (различия в 2-3 знаке — норм).
31. Да, cost должен монотонно падать и выходить на плато.
32. Cost растёт/скачет — слишком большой шаг, перепрыгиваем минимум. Это divergence.
33. `KMeans(n_clusters=3, n_init=10).fit_predict(X)`; раскрась `c=labels`.
34. Локоть на K=3 — после него inertia падает медленно.
35. `model.feature_importances_` — сумма равна 1, больший = важнее.
36. Среднее по 5 разбиениям надёжнее: один split может случайно повезти/не повезти.
37. `df["col"] = df["col"].fillna(df["col"].mean())`
38. 3 колонки (или 2 при drop_first=True).
39. Симметричный "колокол" вокруг нуля без перекоса — модель не ошибается систематически.
40. `Pipeline([("scaler", StandardScaler()), ("model", LinearRegression())])`
41. `y_pred = X @ w + b; dw = (X.T @ (y_pred - y)) / n; db = (y_pred - y).mean()`
42. `(x - x.mean(axis=0)) / x.std(axis=0)` — совпадёт со StandardScaler (он использует std по population).
43. Шаги: случайные центры → расстояния до центров → argmin → новые центры = mean кластеров → повтор.
44. `mask = np.abs(x - x.mean()) > 3 * x.std()`
45. AUC ближе к 1.0 — хорошо; 0.5 — модель как монетка.
46. `GridSearchCV(rf, params, cv=5).fit(X,y); .best_params_`
47. Косинусная близость: `(a @ b) / (norm(a)*norm(b))` по общим оценкам; рекомендуй фильм, который похожий юзер оценил высоко, а целевой не смотрел.
48. load_digits проще: нормализуй на /16, train 10-20 эпох, accuracy > 0.95 достижим.
49. Да, цифры образуют видимые сгустки, хотя и пересекаются.
50. Формула обновления: Q[s,a] += alpha*(reward + gamma*max(Q[s']) − Q[s,a]). Маршрут — идти по argmax Q.
